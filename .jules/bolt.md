## 2025-02-20 - [Vectorizing DP Optimization]
**Learning:** Python loops inside Dynamic Programming backward induction create massive bottlenecks in this specific codebase context. Due to grid evaluation dimensions `O(nodes * V^2)`, standard loop implementations become prohibitively expensive for realistic resolutions (100 nodes x 80 velocity levels).
**Action:** When working on grid-based calculations in DP optimization loops, always aggressively use vectorized operations via numpy matrix broadcasting to perform evaluation. Filtering calculations directly through numpy index masking yields two orders of magnitude in speedup while maintaining physical correctness.
## 2026-06-06 - [CasADi vs NumPy Interpolation]
**Learning:** Functions in this codebase (`VehicleDynamics`) are often used by BOTH the DP optimizer (using NumPy arrays) AND the NLP optimizer (using CasADi symbolic variables like `SX`). Standardizing entirely on `np.interp` or boolean masking will crash CasADi optimization paths because `SX` cannot be implicitly evaluated or converted to numeric matrices inside standard Python conditionals.
**Action:** When vectorizing inner mathematical functions, always maintain separate branches: `if not isinstance(P, np.ndarray):` (which handles floats and CasADi `SX` symbols safely) and the highly optimized NumPy block (which handles array computations).
