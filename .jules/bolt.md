## 2025-02-20 - [Vectorizing DP Optimization]
**Learning:** Python loops inside Dynamic Programming backward induction create massive bottlenecks in this specific codebase context. Due to grid evaluation dimensions `O(nodes * V^2)`, standard loop implementations become prohibitively expensive for realistic resolutions (100 nodes x 80 velocity levels).
**Action:** When working on grid-based calculations in DP optimization loops, always aggressively use vectorized operations via numpy matrix broadcasting to perform evaluation. **Learning:** Using multiple sequential boolean masks (`mask1 = ...`, `eff[mask1] = ...`) to map values in NumPy is significantly slower than using `np.interp` for piecewise linear functions or evaluating the entire array simultaneously.
**Action:** When evaluating piecewise functions on NumPy arrays, initialize X/Y interpolation points and use `np.interp`. To avoid double-evaluating mathematical functions across conditionally split arrays (like positive vs negative bounds), compute the function once over the absolute value of the entire array.
